## 2. 功能详解与本地差距

### 2.1 流式断线 + mid-turn steer（P0）

**上游问题**：桌面聊天在 turn 进行中发跟进消息会被丢弃；流式中断后重连行为错误；steer 在 final answer 阶段插入的消息丢失。

**涉及文件**：

| 层 | 文件 |
|----|------|
| 后端 | `core/agent.py`、`core/reasoning_engine.py`、`api/routes/chat.py`、`api/routes/double_texting.py`、`api/routes/conversation_lifecycle.py` |
| 单测 | `tests/unit/test_steer_done_drain.py`（**新增**）、`test_c17_sse_replay.py`（扩展） |
| 前端 | `views/ChatView.tsx`（发送逻辑、recovery、steer-or-queue 统一） |

**本地适配**：

- `ChatView.tsx` **块级 merge**；保留 `synapse_*` 事件/storage（`synapse_client_id`、`synapse_data_epoch` 等，见 §2.4）。
- STEER 带附件时后端降级 QUEUE（`chat.py`）——与本地附件上传流程兼容，无需改研发云。
- 合入后跑 `pytest tests/unit/test_steer_done_drain.py tests/unit/test_c17_sse_replay.py`。

---

### 2.2 SSE replay 限定当前 turn（P0）

**上游改法**：

- POST `/api/chat` 开头 `_sse_session.begin_turn()`，replay floor 推到当前 max seq。
- **POST 不再** 根据 `Last-Event-ID` replay 上一 turn 尾巴（根因：新问题顶上旧答复）。
- 前端 **不再** 在新 turn POST 带 `Last-Event-ID`；续流走 GET `/api/chat/resume` + `since_seq`。

**本地适配**：`ChatView.tsx` 与 v1.27.20 已合的 message parts 流并存；merge 时对照 upstream `6334848a` 的 SSE 注释块，勿恢复 POST replay。

---

### 2.3 会话列表时间/排序 #628（P0）

**后端**（`sessions.py` + `session.py`）：

- 新增 `_last_activity_ms()`：以**最后一条可见消息**时间为准。
- `list_sessions` 按 `_last_activity_ms` 排序，不再用被读操作污染的 `last_active`。
- `touch()` 仅真实写入消息时调用；查询路径改 `reactivate()` 不改时间。

**前端**（`ChatView.tsx`）：

- 与后端对账时：**非流式会话**直接用后端 `timestamp`，**禁止** `Math.max(local, backend)` 锁死旧脏缓存。
- 流式中会话可保留 `Math.max` 乐观更新。

**本地**：无研发云冲突；`sessions.py` 在 v1.27.20 已改过 parts/history，需 **三路 merge**（本地 ↔ v1.27.20 ↔ v1.27.21）。

---

### 2.4 data_epoch 按 workspace 隔离 #635（P0，品牌化）

**上游**：`STORAGE_KEY_DATA_EPOCH = openakita_data_epoch_${wsTag}`，并 `removeItem("openakita_data_epoch")` 清全局 legacy key。

**本地必须**：

```typescript
const STORAGE_KEY_DATA_EPOCH = `synapse_data_epoch_${wsTag}`;
try { localStorage.removeItem("synapse_data_epoch"); } catch { /* legacy global */ }
```

**禁止**照抄 upstream 的 `openakita_data_epoch_*` 字符串（会导致与 v1.27.20 已统一的 `synapse_*` 不一致、误触发 factory reset 清会话）。

---

### 2.5 STEER 默认 + QUEUE 超时（P1）

- 桌面默认 **STEER**（turn 中跟进注入 ReAct loop）。
- **QUEUE** 单独超时，避免长 turn 误触 6s 错误。
- 前端 `ChatView` 与后端 `double_texting.py` 同步合入。

**本地**：不影响研发云会议室/工单；仅主聊天 `ChatView`。

---

### 2.6 推理链隐藏时空白气泡（P1）

- `MessageBubble.tsx` 在全局 `showChain=off` 且无可渲染正文时，提供「查看过程」手柄展开 `thinkingChain`。

**合入方式**：新增函数 + 块级改 `MessageBubble`/`MessageParts`；与 v1.27.20 parts 投影一致。

### 2.10 站内信 Inbox（P2 可选，默认跳过）

**是什么**：App 内「公告/通知中心」——Topbar 角标 + `InboxView` 列表；后端本地存消息，可从远端拉广播（上游设计也用来发「有新版本」类通知，**但这不是安装包更新本身**）。

**和内部技能 / 研发云**：无关系；和 **Updater** 在上游代码里有关联（updater 请求会带 inbox 的 install_id），**Synapse 不合 upstream updater 时可整包跳过 Inbox**，避免半套功能。

**若将来要做**：单独立项，endpoint 走公司服务，**不要**绑 upstream CDN；`App.tsx` 块级合入，保留 workbench/引导。

**本次默认**：**不合** inbox 模块、`InboxView`、`platform/index.ts` 里 upstream 的 updater header 逻辑。

---

### 2.11 桌面自动更新 Updater（跳过，非本次合入范围）

**是什么**：Tauri 插件定期访问 **endpoint URL** 上的 `latest.json`，发现新版本则下载安装包；**pubkey** 用来验证包没被改过。

**本地现状**（`apps/setup-center/src-tauri/tauri.conf.json`）：

- endpoints：`https://synapse.ai/api/latest.json`、`https://git-nj.iwhalecloud.com/osc/iwhalecloud-agent-skills`
- pubkey：本地已有 minisign 公钥（与 Synapse/公司发版私钥配对）

**v1.27.21 upstream 改了什么**：换成 openakita 官方 CDN + 新 pubkey + `createUpdaterArtifacts` 等——**全部不合**。

**同步规则**：动到 `tauri.conf.json` 时 **只 bump version**，**不要**改 `plugins.updater` 的 endpoints/pubkey。
