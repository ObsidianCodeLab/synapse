## 2. 功能 A：取消后续聊恢复（后端，P0）

### 2.1 问题

v1.28 已落地 `persist/load/clear` 辅助函数，但未接入推理引擎；用户取消后下一轮从扁平文本历史重建，**已完成的 tool 会被重跑**。

### 2.2 上游解法

1. **`cancel_cleanup.py` 新增**
   - `RESUME_HINT_FRESHNESS_SECONDS = 3600`（1h 内才注入「接着干」提示）
   - `persisted_age_seconds()`、`has_tool_blocks()`
   - load 与 hint **解耦**：24h 内始终恢复结构化状态，超过 1h 只恢复不重提示

2. **`TaskState.is_sub_agent`**：子 agent 与父会话共享 `conversation_id`，禁止 persist

3. **`ReasoningEngine` 新增并接线**
   - `_resume_eligible`、`_maybe_persist_cancelled_working_messages`
   - `_maybe_load_resume_working_messages`、`_maybe_clear_resume_state`
   - cancel farewell / stream finally / run wrapper finally 全路径覆盖

4. **`server.py` 启动钩子**：`cleanup_expired_working_messages()`

5. **集成测试** `tests/integration/test_cancel_resume_wiring.py`

### 2.3 本地映射路径

| 上游 | 本地 |
|------|------|
| `src/openakita/core/*.py` | `src/synapse/core/*.py` |
| `src/openakita/api/server.py` | `src/synapse/api/server.py`（**手工 merge**，保留 yuque/gitnexus/dev_iwhalecloud） |
| 测试 import `openakita` | 改为 `synapse` |

### 2.4 与本地定制兼容性

- 不涉及研发云 / 语雀 / GitNexus 路由
- 与现有 `orchestrator._call_agent(is_sub_agent=depth>0)` 天然契合
- `multi_agent_enabled` 常开语义不受影响

## 3. 功能 B：崩溃诊断增强（P1）

### 3.1 问题

heap-corruption fast-fail、WebView2 渲染进程崩溃不走 SEH filter，反馈包仅有自家 `*.dmp`，诊断证据缺失。

### 3.2 上游解法

| 改进 | 说明 |
|------|------|
| WER `ReportQueue` + `ReportArchive` | 扫描 `%LOCALAPPDATA%` 与 `%ProgramData%` |
| WER minidump 预算 | `WER_DUMP_MAX_BYTES = 8MB`，跳过 `*.hdmp` |
| WebView2 Crashpad | `%LOCALAPPDATA%\com.openakita.setupcenter\EBWebView\Crashpad\reports` |
| `crash_handler.rs` | 文档说明 SEH 盲区 |

### 3.3 本地品牌化适配

| 上游 | 本地 |
|------|------|
| 匹配词 `openakita` | **`synapse`** |
| `com.openakita.setupcenter` | **`com.synapse.setupcenter`**（与 `tauri.conf.json` identifier 一致） |
| `_resolve_openakita_home_dir()` | **`_resolve_synapse_home_dir()`** |

## 4. 功能 C：桌面崩溃自愈（P2，高风险合并面）

### 4.1 问题

tao/wry ud2 / fast-fail 直接杀进程，panic hook 与 SEH 均无法恢复窗口。

### 4.2 上游机制

```
主进程(release) ──spawn──► 看门狗 (--watchdog <pid>)
                              ├─ WaitForSingleObject 等父进程
                              ├─ exit-handled 标记 → 不拉起
                              ├─ 30s 冷却（v1.27.18 单点，v1.27.19 改为断路器）
                              └─ --auto-restarted 拉起新实例
```

配套：

- `mark_exit_handled()`：`RunEvent::Exit`、panic self-heal、`prepare_relaunch` Tauri command
- `platform/index.ts`：`relaunchApp()` 前先 `invoke("prepare_relaunch")`
- **仅 release** 启用看门狗（`tauri dev` 不 spawn）
- `crash_handler::install()` 挂载 SEH minidump（本地原先有模块但未接线）

### 4.3 本地 merge 策略（实际采用）

**`main.rs` 相对 v1.27.17 已有 +738 行本地增量** → **禁止整文件覆盖**，块级移植：

| 代码块 | 路径品牌化 |
|--------|------------|
| `exit_handled.marker` | `~/.synapse/` |
| `watchdog-relaunch.marker` | `~/.synapse/` |
| `restart.marker` | `~/.synapse/` |
| `prepare_relaunch` / `spawn_watchdog` / `run_watchdog` | 保留本地 rd_terminal、服务 PID 管理 |
| `try_self_heal_relaunch` | tao#1180 panic 字符串匹配 |

**依赖**：`Cargo.toml` 增加 `windows-sys`（`crash_handler` 编译所需）。

### 4.4 相对上游完整 main.rs 的精简项（已知）

本次为块级适配，**未**整段搬入上游以下增强（不影响看门狗主路径）：

- `spawn_machine_info_collector` / 更丰富 panic hook（backtrace、机器指纹）
- `log_to_file` → `record_event` 转发（SEH dump 旁 events 轨迹）
- `frontend_session_marker` / `take_startup_recovery_notice`

可在后续小 patch 补 `crash_handler::record_event(msg)` 一行。
