## 2. 功能详解与本地差距

### 2.1 sessions.json 严格原子写（P0，#647）

**上游问题**：进程崩溃时 `sessions.json` 可能被半写 corrupt；recovery 路径曾 `unlink` + `rename` 仍有窗口。

**上游改法**：

- `utils/atomic_io.py`：统一 `atomic_json_write`（fsync、备份、无 direct-write fallback）。
- `sessions/manager.py`：`_save_sessions()` 仅通过 `atomic_json_write`；`flush`/`mark_clean` 在失败时恢复 `_dirty=True`。
- 多处在 `safe_json_write` → `atomic_json_write`（orchestrator 子状态等）。

**本地差距**：

- `src/synapse/sessions/manager.py` 的 `_save_sessions` **仍为** 手写 `sessions.json.tmp` + `json.dump` + `replace`（约 985 行起）。
- `_save_channel_registry` 已用 `atomic_json_write`。
- 本地 **PR-N1** 会话 ID 安全校验、invalid_id 采样日志需 **保留**。

**本地适配**：

- **块级**替换 `_save_sessions` 实现为 upstream `atomic_json_write` 路径。
- recovery 段：`temp_file.replace(sessions_file)` 与 upstream 对齐（本地 702–704 行仍 `unlink`+`rename` 的需一并改）。
- `orchestrator._persist_sub_states`：若仍为 `safe_json_write`，改为 `atomic_json_write`（勿动 rd_meeting 委派逻辑）。

**验收**：

```bash
pytest tests/unit/test_json_atomic_migration.py tests/unit/test_session.py -q
```

| 单测 | `test_identity.py`、`test_profile_identity_prompt.py`、`test_prompt_compiler.py`、`test_manager_extra.py` |

**本地差距**：

### 2.3 Setup Center pip 安装 + Tauri 生命周期（P0，高风险）

**上游问题**：

- 后台 worker 在 UI teardown 后仍持有 Tauri handle → 关机崩溃 / 引导安装卡住。
- pip 传 `--resume-retries` 导致旧 pip 安装失败。
- 进度事件从 `pip_install_event` 改为 Rust 侧 **可轮询状态** + 磁盘日志。

**涉及文件**：

| 层 | 文件 |
|----|------|
| Rust | `apps/setup-center/src-tauri/src/main.rs`（**+1100 行**） |
| 前端 | `apps/setup-center/src/App.tsx`（~152 行：轮询、`installId`、`pipInstallPolling`） |
| 工具 | `tools/audit_tauri_lifecycle.py`（**新增**，生命周期审计） |
| 依赖 | `apps/setup-center/package.json`（若有新脚本引用） |

**本地差距**：

- `App.tsx` 仍 `listen("pip_install_event")`；引导安装已 `packageSpec: "synapse"`（upstream 为 `openakita`）。
- `main.rs` 含大量 **本地独有**：`synapse-desktop` 路径、`rd_terminal/`、研发云 devservice、进程清理 grep `synapse.main` 等。

**本地适配（必须块级）**：

1. **App.tsx**：合入 `pipInstallPolling` / `invoke("pip_install_progress")` / `installId` 传参；**保留** workbench、Inbox、`ob-core-agent`、研发云引导链。
2. **main.rs**：按 upstream 合入 `pip_install_progress`、生命周期 guard、pip flag 修复；**禁止**覆盖本地 bundled backend 路径、Synapse 进程管理、rd_terminal 模块。
3. 合入后运行：`python tools/audit_tauri_lifecycle.py`（若脚本合入）。
4. **品牌化**：所有 upstream 文案/命令中的 `openakita` 安装目标改为 **`synapse`**（本地已部分完成，merge 时勿回退）。

**禁止**：对 `main.rs` 做「取 upstream 整文件 + 品牌化」——会删掉 `rd_terminal` 与本地打包逻辑。
